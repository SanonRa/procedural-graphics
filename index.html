Procedural Graphics

Code-as-Assets

An exploration into generating graphics directly from source code. Sprites, caves, terrain and infinite worlds created algorithmically— without relying on traditional art assets.

Explore Demos View on GitHub

Traditional graphics pipelines:

"If it can be mathematically described, it can be procedurally generated."

Inspiration

Inspired by the philosophy behind TempleOS and the work of Terry A. Davis.

Rather than recreating TempleOS, this project explores the idea that graphics and visual assets can emerge directly from algorithms and source code.
